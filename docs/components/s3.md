# S3 Component

The S3 component provides an S3 bucket with CloudFront CDN distribution for serving static assets securely and efficiently.

## Features

- Private S3 bucket with versioning enabled
- CloudFront CDN distribution for global content delivery
- Origin Access Identity (OAI) for secure access
- CORS configuration for cross-origin requests
- Server-side encryption (AES256)
- Optional lifecycle rules for content expiration
- HTTPS-only access via CloudFront

## Usage

### Basic Example

```typescript
import { s3 } from 'infra-foundry'

const bucket = new s3.S3Bucket({
  name: 'my-assets',
})

export const cdnUrl = bucket.assetsCdn.domainName
export const bucketName = bucket.assetBucket.id
```

### With Content Expiration

```typescript
import { s3 } from 'infra-foundry'

const bucket = new s3.S3Bucket({
  name: 'temporary-assets',
  contentExpiration: 30, // Expire content after 30 days
})
```

## Configuration

### S3BucketConfig

| Property            | Type     | Required | Default     | Description                                                 |
| ------------------- | -------- | -------- | ----------- | ----------------------------------------------------------- |
| `name`              | `string` | Yes      | -           | Base name for the S3 bucket and related resources           |
| `contentExpiration` | `number` | No       | `undefined` | Number of days after which content expires (lifecycle rule) |

## Outputs

The `S3Bucket` component provides the following outputs:

### assetBucket

Type: `s3.Bucket`

The S3 bucket resource for storing assets.

```typescript
const bucket = new s3.S3Bucket({ name: 'my-assets' })
console.log(bucket.assetBucket.id) // Bucket name
console.log(bucket.assetBucket.arn) // Bucket ARN
```

### assetsCdn

Type: `cloudfront.Distribution`

The CloudFront distribution for serving assets.

```typescript
const bucket = new s3.S3Bucket({ name: 'my-assets' })
console.log(bucket.assetsCdn.domainName) // CloudFront domain
export const cdnUrl = bucket.assetsCdn.domainName
```

## Features in Detail

### Security

- **Private Bucket**: The S3 bucket is created with private ACL
- **OAI Access**: Only CloudFront can access the bucket via Origin Access Identity
- **HTTPS Only**: CloudFront redirects all HTTP requests to HTTPS
- **Encryption**: Server-side encryption enabled by default (AES256)

### Performance

- **Global CDN**: CloudFront distributes content globally with low latency
- **Caching**: Default TTL of 1 day, max TTL of 1 year
- **IPv6**: IPv6 enabled for modern network support

### CORS Configuration

The bucket includes CORS rules allowing:

- Methods: `GET`, `HEAD`
- Origins: All origins (`*`)
- Headers: All headers (`*`)
- Max age: 3000 seconds

### Lifecycle Management

When `contentExpiration` is set:

- Current versions expire after specified days
- Non-current versions expire after specified days
- Incomplete multipart uploads are aborted after 7 days

## Complete Example

```typescript
import * as pulumi from '@pulumi/pulumi'
import { s3 } from 'infra-foundry'

// Create S3 bucket with CloudFront
const assetsBucket = new s3.S3Bucket({
  name: 'my-app-assets',
  contentExpiration: 90, // Expire old content after 90 days
})

// Export outputs
export const bucketName = assetsBucket.assetBucket.id
export const bucketArn = assetsBucket.assetBucket.arn
export const cdnDomain = assetsBucket.assetsCdn.domainName
export const cdnUrl = pulumi.interpolate`https://${assetsBucket.assetsCdn.domainName}`

// Upload a file to the bucket
import * as aws from '@pulumi/aws'

const indexHtml = new aws.s3.BucketObject('index.html', {
  bucket: assetsBucket.assetBucket.id,
  key: 'index.html',
  source: new pulumi.asset.FileAsset('./public/index.html'),
  contentType: 'text/html',
})
```

## Use Cases

1. **Static Website Hosting**: Serve static websites via CloudFront
2. **Asset Storage**: Store and serve images, videos, and other media
3. **Application Assets**: Host CSS, JavaScript, and other static assets
4. **Temporary Storage**: Use with content expiration for temporary files

## Best Practices

1. **Use CloudFront Domain**: Always use the CloudFront domain for public access, not the S3 bucket URL
2. **Set Appropriate Expiration**: Use `contentExpiration` for temporary content to reduce storage costs
3. **Cache Headers**: Set appropriate cache headers on uploaded objects
4. **Versioning**: The bucket has versioning enabled - use it for rollback capability
5. **Monitoring**: Monitor CloudFront metrics for cache hit ratio and performance

## Troubleshooting

### Issue: Content not updating in CloudFront

CloudFront caches content. To force an update:

```bash
# Invalidate CloudFront cache
aws cloudfront create-invalidation \
  --distribution-id <distribution-id> \
  --paths "/*"
```

### Issue: Access denied when accessing objects

Ensure you're accessing via CloudFront domain, not S3 bucket URL. The bucket is private and only accessible through CloudFront.

### Issue: CORS errors

The default CORS configuration allows all origins. If you need to restrict origins, you'll need to customize the component or create a custom bucket policy.
